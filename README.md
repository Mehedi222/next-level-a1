# TypeScript-এর keyof অপারেটর: এটি কী এবং কেন দরকার?

TypeScript-এ কোড লেখার সময় আমরা প্রায়ই অবজেক্টের প্রপার্টি নিয়ে কাজ করি।
কিন্তু কী হবে যদি আমরা এমন কোনো প্রপার্টির নাম ব্যবহার করার চেষ্টা করি যার অস্তিত্বই নেই? এখানেই `keyof` অপারেটরটি আমাদের সাহায্য করতে আসে।

## keyof অপারেটর আসলে কী?

খুব সহজ ভাষায়, `keyof` একটি **টাইপ অপারেটর** যা একটি অবজেক্ট টাইপ (object type) থেকে তার সমস্ত কী (key) বা প্রপ��র্টির নামের একটি **ইউনিয়ন টাইপ** (union type) তৈরি করে।

# Enums কেন ব্যবহার করবেন?

ধরুন, আপনি একটি অ্যাপ্লিকেশনে ব্যবহারকারীর বিভিন্ন ভূমিকা (user role) নিয়ে কাজ করছেন। আপনি হয়তো এভাবে সংখ্যা ব্যবহার করতে পারেন:

- `0` মানে `ADMIN`  
- `1` মানে `EDITOR`  
- `2` মানে `USER`

কোডের মধ্যে হুট করে `if (user.role === 1)` লেখা দেখলে বোঝা কঠিন যে `1` মানে আসলে কী।  
এই অস্পষ্ট সংখ্যাগুলোকে **"ম্যাজিক নাম্বার"** (magic numbers) বলা হয়।

`enum` আমাদের এই "ম্যাজিক নাম্বার" বা "ম্যাজিক স্ট্রিং" পরিহার করতে সাহায্য করে এবং কোডকে আরও  
পাঠযোগ্য (readable) ও রক্ষণাবেক্ষণযোগ্য (maintainable) করে তোলে।

TypeScript-এ প্রধানত দুই ধরনের enum রয়েছে:

1. Numeric Enums (সংখ্যাভিত্তিক)
2. String Enums (স্ট্রিংভিত্তিক)

---

## ১. Numeric Enums (সংখ্যাভিত্তিক এনাম)

Numeric enums হলো সবচেয়ে প্রচলিত। এখানে প্রতিটি সদস্যের একটি সংখ্যাসূচক মান থাকে।

আপনি যদি কোনো মান নির্দিষ্ট না করে দেন, তবে enum ডিফল্টভাবে `0` থেকে শুরু হয় এবং  
প্রতিটি সদস্যের মান এক এক করে বাড়তে থাকে।

### উদাহরণ: ব্যবহারকারীর ভূমিকা (User Role)

```ts
enum UserRole {
  ADMIN,  // মান পাবে 0
  EDITOR, // মান পাবে 1
  USER    // মান পাবে 2
}

// এটি ব্যবহার করা
const currentUserRole: UserRole = UserRole.ADMIN;

if (currentUserRole === UserRole.ADMIN) {
  console.log("ব্যবহারকারী একজন অ্যাডমিন।");
}

console.log(UserRole.EDITOR); // আউটপুট: 1
